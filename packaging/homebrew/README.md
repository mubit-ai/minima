# Homebrew distribution

`minima` (the TUI coding agent) installs via Homebrew from the `mubit-ai/homebrew-minima` tap.
As of v0.5.0 the formula ships the **prebuilt TypeScript/Bun binary** (`packages/tui`), not the
old Python virtualenv build — so install is a download + extract (seconds) instead of a ~5-minute
wheel/venv build, with no Python or runtime dependencies.

> The Python `minima-cli` package still publishes to **PyPI** (the `minima_client` SDK + server
> tooling); only the brew-installed CLI is the binary. See `../../.github/workflows/prod.yml`
> (PyPI) and `release-cli.yml` (binaries + tap bump).

## Tap layout

```
github.com/mubit-ai/homebrew-minima
└── Formula/
    └── minima.rb     # rendered by render_formula.py at each release
```

Users install with:

```bash
brew install mubit-ai/minima/minima
# or:  brew tap mubit-ai/minima && brew install minima
```

## How a release updates the formula (automated)

On a published GitHub release, `.github/workflows/release-cli.yml`:

1. Cross-compiles the binary for macOS (arm64 + x64, ad-hoc codesigned on a macOS runner) and
   Linux (x64 + arm64) via `packages/tui/scripts/build-all.ts`.
2. Tars each as `minima-<version>-<os>-<arch>.tar.gz` and attaches them to the release.
3. Runs `render_formula.py --version <v> --dist <dir>` to emit the binary formula (per-platform
   `url` + `sha256` computed from the actual tarballs + `bin.install "minima"`), then opens a PR
   to the tap. `brew test-bot` (tap `tests.yml`) validates install + `test do` before merge.

## Rendering the formula manually

```bash
# after building tarballs into <dir> (e.g. packages/tui/dist):
python packaging/homebrew/render_formula.py --version 0.5.0 --dist packages/tui/dist --out Formula/minima.rb
```

## History

The pre-0.5.0 Python formula vendored ~50 per-arch wheels generated by `gen_resources.py` (removed);
that machinery is obsolete now that the CLI is a single self-contained binary.
